# Инструкция по установке киоска меню для столовой «Фаберже»

## 📋 Описание

Этот проект предназначен для автоматической установки и настройки системы отображения меню на двух мониторах с разным разрешением под управлением **Ubuntu Server 24.04**.

Система автоматически:
- Устанавливает все необходимые компоненты (веб-сервер, браузер, оконный менеджер)
- Настраивает автоматический вход в систему
- Запускает два окна браузера в режиме киоска на разных мониторах
- Отображает меню столовой с автоматическим обновлением

---

## 🖥️ Требования

- **Операционная система**: Ubuntu Server 24.04
- **Два монитора** с разным разрешением (по умолчанию: 1280x1024 и 1920x1080)
- **Подключение к интернету** (для установки пакетов и клонирования репозитория)
- **Права sudo** для запуска скрипта установки

---

## 🚀 Быстрый старт

### Шаг 1: Подготовка системы

1. Установите Ubuntu Server 24.04 на ваш компьютер/сервер
2. Зарегистрируйтесь в системе под обычным пользователем (не root)
3. Убедитесь, что у вас есть подключение к интернету

### Шаг 2: Скачивание скрипта установки

Откройте терминал и выполните команды:

```bash
# Перейдите в домашнюю директорию
cd ~

# Скачайте скрипт установки
wget https://raw.githubusercontent.com/cyb3rL1ght/Faberge/master/install_kiosk.sh

# Или склонируйте весь репозиторий
git clone https://github.com/cyb3rL1ght/Faberge.git
cd Faberge
```

### Шаг 3: Запуск установки

**Важно!** Скрипт необходимо запускать от имени root через `sudo`:

```bash
sudo ./install_kiosk.sh
```

или если вы скачали репозиторий:

```bash
sudo ./Faberge/install_kiosk.sh
```

### Шаг 4: Перезагрузка

После завершения установки скрипт предложит перезагрузить систему:

```bash
sudo reboot
```

После перезагрузки система автоматически:
- Выполнит вход под вашим пользователем
- Запустит графическую сессию
- Откроет два окна Chromium в режиме киоска

## ⚙️ Настройка

### Изменение разрешения мониторов

По умолчанию установлены следующие разрешения:
- **Левый монитор**: 1280x1024
- **Правый монитор**: 1920x1080

Если ваши мониторы имеют другое разрешение, отредактируйте скрипт установки **перед запуском**:

```bash
nano install_kiosk.sh
```

Найдите строки (примерно 17-20):

```bash
SCREEN_0_W=1280   # Ширина левого экрана
SCREEN_0_H=1024   # Высота левого экрана
SCREEN_1_W=1920   # Ширина правого экрана
SCREEN_1_H=1080   # Высота правого экрана
```

Измените значения на нужные вам и сохраните файл (Ctrl+O, Enter, Ctrl+X).

### Настройка после установки

Если мониторы отображаются неправильно после установки, настройте их вручную:

1. Узнайте имена ваших видеовыходов (HDMI-1, DP-1 и т.д.):
```bash
DISPLAY=:0 xrandr
```

2. Настройте расположение мониторов (пример для HDMI-1 и HDMI-2):
```bash
DISPLAY=:0 xrandr --output HDMI-1 --mode 1280x1024 --pos 0x0 --rotate normal \
       --output HDMI-2 --mode 1920x1080 --pos 1280x0 --rotate normal
```

Где:
- `--output` - имя видеовыхода
- `--mode` - разрешение экрана (ширина x высота)
- `--pos` - позиция экрана (X x Y), где X - отступ слева, Y - отступ сверху
- `--rotate normal` - нормальная ориентация (без поворота)

3. Чтобы настройки применялись автоматически при каждой загрузке, добавьте команду в файл `.xinitrc`:
```bash
nano ~/.xinitrc
```

Добавьте строку с `xrandr` после строки `openbox-session &`.

## 🍽️ Редактирование меню

Для изменения позиций в меню отредактируйте файл `menu.csv`:

```bash
sudo nano /var/www/faberge/menu.csv
```

Файл имеет следующую структуру:
```
category,name,price,description
salads,Цезарь,150,Классический салат с курицей
soups,Борщ,120,Традиционный украинский борщ
```

**Категории:**
- `salads` - Салаты
- `mains1`, `mains2` - Вторые блюда (две колонки)
- `soups` - Первые блюда
- `sides` - Гарниры
- `drinks` - Напитки

После редактирования сохраните файл (Ctrl+O, Enter, Ctrl+X). Изменения применятся автоматически в течение 5 минут (или сразу после перезагрузки страницы).

## 🔧 Полезные команды

### Управление веб-сервером (Nginx)

```bash
# Перезапустить Nginx
sudo systemctl restart nginx

# Проверить статус Nginx
sudo systemctl status nginx

# Просмотреть логи ошибок
sudo tail -f /var/log/nginx/error.log

# Проверить конфигурацию
sudo nginx -t
```

### Управление графической сессией

```bash
# Перезапустить графическую сессию
pkill xinit && startx

# Выйти из графической сессии
# Нажмите: Ctrl+Alt+Backspace

# Переключиться на консоль (терминал)
# Нажмите: Ctrl+Alt+F2 (или F3, F4...)

# Вернуться в графическую сессию
# Нажмите: Ctrl+Alt+F1
```

### Диагностика

```bash
# Проверить запущенные процессы Chromium
ps aux | grep chromium

# Проверить настройку дисплеев
xrandr

# Проверить открытые окна
wmctrl -l

# Перезагрузить систему
sudo reboot
```

## ❓ Решение проблем

### Проблема: Система не загружается в графический режим

**Решение:**
1. Нажмите Ctrl+Alt+F2 для перехода в консоль
2. Залогиньтесь под своим пользователем
3. Проверьте логи:
   ```bash
   cat ~/.xsession-errors
   ```
4. Попробуйте запустить вручную:
   ```bash
   startx
   ```

### Проблема: Мониторы показывают не то изображение

**Решение:**
1. Узнайте имена видеовыходов:
   ```bash
   DISPLAY=:0 xrandr
   ```
2. Настройте мониторы вручную (замените HDMI-1 и HDMI-2 на ваши):
   ```bash
   DISPLAY=:0 xrandr --output HDMI-1 --mode 1280x1024 --pos 0x0 \
          --output HDMI-2 --mode 1920x1080 --pos 1280x0
   ```
3. Чтобы настройки применялись автоматически, добавьте эту команду в файл `~/.xinitrc` после строки `openbox-session &`

### Проблема: Браузер не открывается или показывает ошибку

**Решение:**
1. Проверьте, работает ли Nginx:
   ```bash
   sudo systemctl status nginx
   ```
2. Проверьте, доступны ли файлы меню:
   ```bash
   curl http://localhost/menu-left.html
   ```
3. Перезапустите Nginx:
   ```bash
   sudo systemctl restart nginx
   ```

### Проблема: Курсор мыши виден на экране

**Решение:**
Курсор должен скрываться автоматически через 0.1 секунды. Если этого не происходит:
1. Проверьте, установлен ли unclutter:
   ```bash
   which unclutter
   ```
2. Запустите вручную:
   ```bash
   unclutter -idle 0.1 -root &
   ```

### Проблема: Окна не позиционируются правильно

**Решение:**
1. Проверьте, что wmctrl установлен:
   ```bash
   which wmctrl
   ```
2. Проверьте заголовки окон:
   ```bash
   wmctrl -l
   ```
3. Убедитесь, что заголовки совпадают с теми, что указаны в `.xinitrc`

## 🔄 Обновление системы

### Обновление файлов меню из репозитория

```bash
# Перейдите в директорию репозитория
cd ~/Faberge

# Получите последние изменения
git pull

# Скопируйте обновленные файлы в веб-директорию
sudo cp -r ~/Faberge/* /var/www/faberge/

# Установите правильные права
sudo chown -R www-data:www-data /var/www/faberge
```

### Обновление скрипта установки

Если вышла новая версия скрипта установки:

```bash
# Скачайте новую версию
wget -O ~/install_kiosk_new.sh https://raw.githubusercontent.com/cyb3rL1ght/Faberge/master/install_kiosk.sh

# Сделайте исполняемым
chmod +x ~/install_kiosk_new.sh

# Запустите (он обновит существующую установку)
sudo ~/install_kiosk_new.sh
```

## 💡 Советы для начинающих

### Что такое терминал?
Терминал (консоль, командная строка) — это текстовый интерфейс для ввода команд компьютеру. В Ubuntu он открывается сочетанием клавиш `Ctrl+Alt+T`.

### Как скопировать и вставить в терминале?
- **Копировать**: `Ctrl+Shift+C` (или правая кнопка мыши → Копировать)
- **Вставить**: `Ctrl+Shift+V` (или правая кнопка мыши → Вставить)

### Что такое sudo?
`sudo` — это команда для выполнения действий от имени администратора (root). Когда вы видите `sudo` перед командой, система попросит вас ввести пароль.

### Как выйти из редактора nano?
1. Нажмите `Ctrl+X`
2. Если нужно сохранить, нажмите `Y` (Yes), затем `Enter`
3. Если не нужно сохранять, нажмите `N` (No)

### Что делать, если что-то пошло не так?
Не паникуйте! В Linux почти всё можно исправить:
1. Перезагрузитесь: `sudo reboot`
2. Если не помогает, попробуйте восстановить систему из резервной копии
3. Обратитесь за помощью на форум или создайте issue на GitHub
