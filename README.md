## 🚀 Назначение

Настроить неттоп 💻 с Ubuntu Server в режиме киоска 🕹️ для отображения двух независимых веб-страниц на двух внешних мониторах. 🖥️🖥️

Система автоматически:
- ✨ Запускает два окна Chromium в режиме киоска
- 🖱️ Скрывает курсор мыши
- 🔌 Отключает скринсейвер и энергосбережение
- 📶 Создаёт Wi-Fi точку доступа для клиентов

---

## 🛠 Требования

### Оборудование:
- 💻 1 x Неттоп (любой x86_64 компьютер)
- 🧠 ОЗУ: не менее 4GB (рекомендуется 8GB+)
- 💾 SSD/HDD: не менее 32GB свободного места
- ⌨️ 1 x USB-клавиатура
- 🖱️ 1 x USB-мышь (по желанию, для начальной настройки)
- 🖥️ 2 x HDMI/DisplayPort/VGA монитора
- 🔌 Соответствующие кабели для подключения мониторов
- 🔌 1 x Блок питания
- 📶 Wi-Fi адаптер (желательно с поддержкой AP-режима, например на чипе RTL8812AU)

### Программное обеспечение:
- 🐧 Ubuntu Server 22.04 LTS или новее (64-bit)
- 👤 Пользователь с правами sudo

---

## 📋 Подготовка системы

1. 📥 Скачайте образ Ubuntu Server с официального сайта: https://ubuntu.com/download/server
2. 💿 Запишите образ на USB-флешку используя:
   - [Rufus](https://rufus.ie/) (Windows)
   - [Balena Etcher](https://www.balena.io/etcher/) (Windows/macOS/Linux)
   - `dd` (Linux/macOS)
3. 💾 Вставьте флешку в неттоп и загрузитесь с неё
4. 🔄 Следуйте инструкциям установщика Ubuntu Server
5. 🔐 Задайте имя пользователя и пароль (запомните их!)

---

## 🔧 Настройка неттопа

1. 💿 После установки загрузитесь в систему
2. 🌐 Подключите интернет (Ethernet или Wi-Fi)
3. 📦 Обновите систему:
   ```bash
   sudo apt update && sudo apt upgrade -y
   ```
4. 🛠️ Установите git (если не установлен):
   ```bash
   sudo apt install -y git
   ```

---

## 🌐 Доступ к неттопу

Если вы настраиваете удалённо:

1. 📡 Узнайте IP-адрес неттопа:
   ```bash
   ip a
   ```
2. 🔌 Подключитесь по SSH с вашего компьютера:
   ```bash
   ssh ваш_пользователь@IP_АДРЕС
   ```
3. 🔐 Смените пароль если используете стандартный:
   ```bash
   passwd
   ```

## 🎨 Настройка KIOSK-режима

### 1️⃣ Клонирование репозитория

```bash
cd ~
git clone https://github.com/cyb3rL1ght/Faberge.git
cd Faberge
```

### 2️⃣ Настройка скрипта установки

Откройте файл `install_kiosk.sh` и измените следующие переменные под вашу систему:

```bash
nano install_kiosk.sh
```

**Обязательно измените:**

| Переменная | Описание | Пример |
|------------|----------|--------|
| `SCREEN_0_W` | Ширина левого монитора | `1920` |
| `SCREEN_0_H` | Высота левого монитора | `1080` |
| `SCREEN_1_W` | Ширина правого монитора | `1920` |
| `SCREEN_1_H` | Высота правого монитора | `1080` |
| `WIFI_IFACE` | Имя вашего Wi-Fi интерфейса | `wlan0` или `wlx...` |
| `SSID` | Название Wi-Fi сети | `MY_KIOSK_WIFI` |
| `WIFI_PASS` | Пароль Wi-Fi сети | `MySecurePassword123` |

💡 **Как узнать имя Wi-Fi интерфейса:**
```bash
ip link show
# или
iwconfig
```

### 3️⃣ Запуск установки

```bash
chmod +x install_kiosk.sh
sudo bash install_kiosk.sh
```

⏳ Процесс установки займёт 5-15 минут в зависимости от скорости интернета.

### 4️⃣ Перезагрузка

```bash
sudo reboot
```

После перезагрузки система автоматически:
- ✅ Залогинится под вашим пользователем
- ✅ Запустит графическую сессию
- ✅ Откроет два окна Chromium на разных мониторах

## 🧪 Тестирование

1. 🕹️ После перезагрузки проверьте, что на обоих мониторах отображаются веб-страницы
2. 🖱️ Курсор мыши должен быть скрыт
3. ⌨️ Попробуйте нажать F11 или Alt+F4 — они должны быть заблокированы
4. 📶 Проверьте, что Wi-Fi точка доступа видна и к ней можно подключиться

## ⚙️ CSS-переменные для настройки

Вы можете легко кастомизировать внешний вид меню через CSS-переменные в файле `styles.css`:

### 🎨 Цвета и фон

```css
--bg-color: #1a1a2e;          /* Основной цвет фона */
--card-bg: rgba(255,255,255,0.1); /* Фон карточек товаров */
--text-color: #ffffff;         /* Цвет текста */
--accent-color: #e94560;       /* Акцентный цвет (цены, кнопки) */
--combo-bg: linear-gradient(...); /* Фон для КОМБО */
--set-bg: linear-gradient(...);   /* Фон для НАБОРОВ */
```

### 🔤 Шрифты

```css
--title-font-size: 8vh;        /* Размер заголовков */
--price-font-size: 7vh;        /* Размер цены (общий) */
--combo-price-font-size: 7vh;  /* Размер цены в КОМБО */
--set-price-font-size: 7vh;    /* Размер цены в НАБОРАХ */
--description-font-size: 4vh;  /* Размер описания */
```

### 📐 Размеры

```css
--card-padding: 2vh;           /* Отступы внутри карточки */
--gap-size: 3vh;               /* Расстояние между карточками */
```

## 🍔 Формат menu.csv

Файл `menu.csv` содержит данные о товарах. Формат:

```csv
цель;Категория;Наименование;Порция;Цена
```

### Пример:

```csv
комбо;Салаты;ОЛИВЬЕ;1/100;180
набор 2;Салаты;РЫБНЫЙ;1/100;180
набор 2;Вторые блюда;РЫБА ЗАПЕЧЁННАЯ С СОУСОМ ТАР-ТАР;1/130;220
```

### Правила:

- ✅ Разделитель — точка с запятой `;`
- ✅ Первая колонка: цель (`КОМБО`, `НАБОР`, или любая другая при учете добавления фильтра в js)
- ✅ Вторая колонка: Категория
- ✅ Третья колонка: Наименование блюда
- ✅ Четвёртая колонка: граммы на порцию (число)
- ✅ Пятая колонка: цена (число)
- ✅ Карточки `КОМБО` отображаются на левом мониторе (максимально 1 карточка)
- ✅ Карточки `НАБОР` отображаются на правом мониторе  (максимально 2 карточки)

## ❓ Troubleshooting

### 🖥️ Экраны чёрные или не запускаются

Проверьте логи X11:
```bash
cat ~/.xsession-errors
```

Проверьте, запущен ли Chromium:
```bash
ps aux | grep chromium
```

### 📶 Wi-Fi точка доступа не работает

1. Проверьте имя интерфейса:
   ```bash
   ip link show
   ```
2. Убедитесь, что адаптер поддерживает AP-режим:
   ```bash
   iw list | grep -A 10 "Supported interface modes"
   ```
3. Перезапустите службы:
   ```bash
   sudo systemctl restart hostapd dnsmasq
   ```

### 🔄 Окна не позиционируются правильно

1. Проверьте заголовки окон в HTML:
   ```html
   <title>Menu_Left</title>   <!-- для левого -->
   <title>Menu_Right</title>  <!-- для правого -->
   ```
2. Убедитесь, что `wmctrl` установлен:
   ```bash
   which wmctrl
   ```

### 🌐 Страницы не загружаются

Проверьте nginx:
```bash
sudo systemctl status nginx
sudo nginx -t
```

Проверьте наличие файлов:
```bash
ls -la /var/www/faberge/
```

### 🔊 Сброс настроек

Чтобы вернуть всё как было до установки:
```bash
sudo systemctl disable getty@tty1.service
rm -f ~/.xinitrc
rm -f ~/.profile  # или удалите блок автозапуска вручную
sudo systemctl stop hostapd dnsmasq
sudo systemctl disable hostapd dnsmasq
```
